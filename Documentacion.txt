Este proyecto se construye en 3 partes:

*Framework: Es un esquema o marco de trabajo que ofrece una estructura base para elaborar un proyecto con objetivos específicos, una especie de plantilla que sirve como punto de partida para la organización y desarrollo de software.

Definición framework fuentes: https://unirfp.unir.net/revista/ingenieria-y-tecnologia/framework

Frontend: Se utiliza el framework de React, aunque bueno, React como tal no es un framework completo, vendria más a ser una libreria muy completa de JavaScript, la expresión vendría a ser una libreria que se usa como framework. Utilicé Vite como herramienta de compilación, se enfoca en velocidad y enfoque debido a algo llamado Hot Module Reload (HRM Update) que basicamente son actualizaciones del navegador al instanto sin necesidad de recargar la página, solo guardando los cambios del script, esta herramienta tiene multiples frameworks a instalar como React, Svelte, Preact, Vue, etc.

*API: Las API (Application Programming Interface) son mecanismos que permiten a dos componentes de software comunicarse entre sí mediante un conjunto de definiciones y protocolos. 

Definicion API's fuentes: https://aws.amazon.com/es/what-is/api

Backend: En este apartado se implementara una herramienta llamada Node.js que es un motor potente de Javascript capaz de crear o conectar API's, es un entorno de ejecución que permite ejecutar Javascript fuera del navegador,es decir, en el lado del servidor,usé esta herramienta más que nada para unificar al mismo lenguaje que uso para el frontend, además porque viene integrado con la herramienta npm (node package manager) que gestiona paquetes al igual que lo que utiliza react porque npm es un gestor de paquetes para Javascript

Base de datos: Para el llenado de datos que es la parte principal del proyecto. Usaré una biblioteca llamada Faker, es una biblioteca que crea registros falsos indicandole los campos que uno quiera, su creacion de datos es realista y sigue la coherencia de los datos gracias a la estructura que se le impone, estaba pensando en esta opcion en un principio aunque también pensé en JSONPlaceHolder que es un servicio de API's falsas o de testing, sin embargo puede no llegar a respetar la coherencia de los datos porque se planea que la inserción de datos sea moderada, minimo 500 registros, para esto se usará un script python y usando la inserción de datos gracias a la conexión y uso de pymongo como libreria de mongodb en python, asi usando los conocimientos adquiridos en la clase
